# HeadlessDCGO.Engine Goal 상세 지시서 템플릿

## 목적

Goal CSV의 한 줄 설명만으로는 실제 작업을 안정적으로 수행하기 어렵다. 이 템플릿은 각 Goal을 실행 가능한 상세 지시서로 확장하기 위한 기준이다.

각 Goal 상세 지시서는 `docs/goal-specs/<GOAL_ID>.md`에 둔다.

## 필수 섹션

각 Goal 상세 지시서는 아래 섹션을 반드시 포함한다.

### 1. Goal 식별 정보

- Goal ID
- Phase
- 영역
- 목표 이름
- 우선순위
- 선행 Goal

### 2. 목표 설명

이 Goal이 왜 필요한지, 어떤 기준을 닫는지 설명한다.

### 3. 작업 범위

이번 Goal에서 해야 하는 작업만 적는다.

### 4. 범위 밖 작업

이번 Goal에서 하면 안 되는 작업을 명확히 적는다.

### 5. 참조 파일

작업자가 먼저 읽어야 하는 설계 문서, Headless 파일, AS-IS 원본 파일을 적는다.

### 6. 산출물

생성/수정되어야 하는 파일이나 문서를 적는다.

### 7. 단위테스트 요구사항

테스트해야 하는 항목, 테스트 파일 위치, 테스트 명령을 적는다.

### 8. 결과 문서

Goal 완료 시 작성해야 하는 결과 문서 경로와 포함 항목을 적는다.

### 9. 완료 체크리스트

완료 선언 전에 확인해야 할 항목을 체크리스트로 적는다.

### 10. 실행 프롬프트

해당 Goal을 바로 맡길 수 있는 프롬프트를 적는다.

## 중요 원칙

- CSV는 Goal 색인이다.
- 실제 작업 지시는 Goal 상세 지시서를 기준으로 한다.
- Goal 상세 지시서가 없으면 해당 Goal은 실행 준비가 된 것이 아니다.
- 단위테스트와 결과 문서가 없으면 Goal은 완료가 아니다.
