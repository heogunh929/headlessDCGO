## 문제 (AS-IS)

- `HeadlessGameLoop.StepAsync`는 action 1개 처리 후 `EffectScheduler.ResolveAllAsync`만 호출했고, AS-IS의 **반복 처리 루프(룰 처리/자동효과/공격 진행/턴종료 교차 반복)가 통째로 없었다.**
- 공격 state 진행, 룰 처리, 트리거 수집이 step 사이에서 일어나지 않았다.

## 구현 요약

- `GameFlowProcessor.RunToStableAsync(context, ct)`를 추가했다. AS-IS coroutine의 yield/재진입을 **"pending choice면 중단, 다음 step에서 재개"** 로 매핑한다.
  - 루프 진입 시 `ChoiceController.Current.IsPending`이면 즉시 `FlowProcessResult.Paused` 반환.
  - 각 반복: RuleProcess(hook) → AutoProcess(`EffectScheduler.ResolveAllAsync`, 실제 해결) → AttackAdvance(hook, `AttackController.Current.IsPending`일 때) → EndTurnCheck(hook).
  - 진행이 없으면 안정 상태로 종료, terminal이면 종료, `MaxIterations`(256) 가드.
- 이번 골격에서 **활성 단계는 AutoProcess뿐**이며 rule/attack/end는 `false`를 반환하는 hook이다. 각각 G3.5-007 / G3.5-005 / G3.5-008에서 채운다.
- `FlowProcessResult`(Status=Stable/PausedForChoice, ProgressedAny, ResolvedEffectCount, Iterations)를 도입했다.
- `HeadlessGameLoop.StepAsync`가 `EffectScheduler.ResolveAllAsync` 직접 호출 대신 `GameFlowProcessor.RunToStableAsync`를 호출하도록 이식했다. step의 `ResolvedEffectCount`는 루프 누적 해결 수로 보고하고, choice pause 시 trace/메시지를 남긴다.
