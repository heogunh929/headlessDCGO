## 구현 요약

- `MatchStateMutationSink : IEffectMutationSink`를 추가했다. `ICardInstanceRepository`(런타임 권위 저장소) 메타데이터에 직접 쓴다.
- Phase 3.5 범위에 맞춰 대표 mutation kind를 boolean 메타데이터 플래그로 매핑했다:
  - `GrantBlocker` → `hasBlocker` (이미 `BlockTiming`이 소비)
  - `GrantRush` → `hasRush` (이미 `AttackPermanentAction`이 소비)
  - `ScheduleRebootUnsuspend` → `scheduleRebootUnsuspend`
  - `PreventBattleDeletion` → `preventBattleDeletion`
  - `SetSecurityCheck` → `pendingSecurityCheck`
- 대상 카드는 mutation `values["targetEntityId"]`로 해석하고, 없으면 `SourceEntityId`로 폴백한다.
- 미지원 kind는 예외를 던지지 않고 `Unsupported`에 기록 + `ILogSink.Warn` 로깅하여 해결 흐름을 막지 않는다.
- 저장소에 없는 대상은 `Skipped`로 기록 + 로깅한다.
- `CardEffectSchedulerResolver`가 sink 종류에 따라 결과 values에 `appliedMutationCount`/`unsupportedMutationCount`/`skippedMutationCount`(production) 또는 `mutationCount`(recording)를 노출하도록 했다.
- `EngineContext.CreateDefault`가 `CardInstanceRepository`/`LogSink`를 공유하는 `MatchStateMutationSink`를 resolver sink factory로 주입하도록 배선했다.
